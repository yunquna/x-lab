# 🧪 YQNUS-X-Lab 内部实验室

**YQNUS-X-Lab** 是我们专属的内部小工具、前端实验室以及脚本分享中台。本项目包含完整的教学资源、功能探索、以及开箱即用的测试环境，旨在赋能非研发人员自助解决日常工作痛点并大幅提升产出效率。

🌐 **[进入 X-Lab 实验室首页](https://yunquna.github.io/x-lab/)**

---

## 🎯 实验室核心板块

实验室目前划分为三大专区，全部通过我们基于 VitePress 搭建的高速文档引擎驱动：

### 1. 📖 教学与规范指南 (Tutorials)
这里存放着各类内部操作教程与开发规范，带您零基础入门日常效率工具体系。
- **油猴脚本 (Tampermonkey) 入门指南**：图文教您如何运用 AI 工具，几分钟内生成生产级别的增强脚印。
- **数字叉车 / 物流操作引导**：配套图文、案例，教您如何低门槛解决繁冗的数据流转问题。

### 2. 🛠 内部实用小工具集 (Tools)
经过验证并直接可用的独立小工具与插件源码归档，可供研究学习和独立部署。
- **Logistics Table / Table Exporter**：各类进出口物流填表提效、一键导出的成熟插件实例（直接由原项目重构剥离入库）。

### 3. 🕹 极客试验场 (Playground)
一个真正可操作的“实弹”交互演练沙盒。
- **交互靶场与在线编辑器：** 让您可以抛开控制台等繁琐环境，直接在网页的文本框中构建和编辑脚本。
- **安全复制下发方案：** 通过一键预置格式化导出，即使毫无代码根基的人也能跟着引导弹窗两步激活内测工具！

## 🚀 目录结构与本地开发

本项目基于 [VitePress](https://vitepress.dev/) 搭建生成。如果您需要为实验室贡献新的文章内容、增添新的工具分类，可以参考如下规范与指令：

### 环境依赖
- **Node.js**: v18+ 及以上版本

### 运行开发
```bash
# 1. 安装项目所有依赖
npm install

# 2. 启动本地热重载调试服务器 (Local Dev)
npm run docs:dev

# 3. 访问输出的控制台地址即可实时预览界面变动
```

## 🔄 自动化部署流 (CI/CD)

实验室仓库现已完全接管全自动化持续集成生命周期：
只要您将各类规范、脚本改动推送到 Github 官方仓库的 `main` 主分支，内置由 GitHub Actions 驱动的部署流水线将会全自动获取最新源码、激活 VitePress 编译构建出纯静态产物资源（Artifacts），并在不足两分钟内无缝上载发行到 GitHub Pages 上。所有内部的受众刷新网页即可获取您的最新工具支持！
